### 第一阶段：工具使用层（15小时，立刻见效）

这部分是**最高杠杆**的学习，今天学完明天就能用。

| 学习内容             | 具体操作                                            | 时间 |
| :------------------- | :-------------------------------------------------- | :--- |
| **Cursor深度使用**   | 快捷键、Composer、Agent模式、上下文引用             | 4h   |
| **Prompt工程基础**   | 写好指令的结构（角色+任务+格式+约束），Few-shot示例 | 3h   |
| **AI辅助编码工作流** | 用AI写单元测试、重构、解释遗留代码、生成文档        | 4h   |
| **AI辅助调试**       | 把错误栈贴给AI分析、让AI解释复杂日志                | 2h   |
| **AI辅助代码审查**   | 让AI review你的PR、发现潜在bug                      | 2h   |

**产出**：日常开发效率提升30-50%，能跟同事分享AI使用技巧。

### 第二阶段：概念理解层（10小时，能面试聊）

不需要手写Agent，但能听懂、能聊、能判断是否适用。

| 学习内容          | 掌握到什么程度                                               |
| :---------------- | :----------------------------------------------------------- |
| **LLM基本原理**   | Token、Temperature、上下文窗口、幻觉、RAG（不用会实现）      |
| **Agent核心概念** | ReAct模式、Tool Calling、Memory是什么，能举例说明            |
| **RAG是什么**     | 知道它解决"私域知识问答"问题，了解大致流程（检索→增强→生成） |
| **主流工具认知**  | 听说过LangChain、Dify、Coze，知道它们分别解决什么问题        |
| **AI项目的边界**  | 什么时候用AI靠谱、什么时候不靠谱，能和PM/产品对齐预期        |

**产出**：技术方案评审时能提出AI相关建议，能评估AI能力的可行性和风险。

### 第三阶段：轻量实践层（15小时，简历可写）

做1-2个小而完整的事情，面试时有据可讲。

**实践A：AI辅助的代码生成工具**

- 写一个脚本，调用AI API自动生成单元测试
- 技术栈：Java + HTTP调用（OpenAI/DeepSeek API）+ JUnit
- 简历写法："开发了AI辅助测试生成工具，减少30%手动编写测试的工作量"

**实践B：Java项目中的RAG体验**

- 用Spring AI或LangChain4j，搭一个简单的"代码库问答"
- 把你的项目文档/注释作为知识库，问"这个模块的入口在哪里"
- 简历写法："在项目中引入RAG能力，帮助新成员快速理解代码结构"

**实践C：智能代码审查助手**

- 写一个脚本，用AI review Git diff
- 输出格式化的审查意见
- 简历写法："构建AI代码审查辅助工具，在CI流程中自动发现潜在问题"

## 需要放弃的内容

以下内容**不需要学**，除非你特别感兴趣：

| 不需要学          | 原因                                    |
| :---------------- | :-------------------------------------- |
| 手写ReAct循环     | 框架帮你做了，理解概念即可              |
| LangChain深层源码 | 加分项不需要造轮子                      |
| 向量数据库调优    | 那是AI工程师的事，你只需要知道用哪个    |
| 模型微调          | 除非你在做垂直领域                      |
| 多Agent协调框架   | 太深了，普通项目用不上                  |
| Python AI生态深入 | 你会Java就能用Spring AI，Python不是必须 |

## 面试话术准备

当被问到"你怎么看AI对开发的影响"时，你可以这样说：

> "我过去6年主要做Java后端，AI出现后，我主动把它融入了日常工作。**现在我的习惯是：先用AI生成架构草稿和单元测试，然后我负责审查、优化和集成**。我发现在做代码重构和历史遗留系统理解时，AI帮助特别大。**我的观点是：AI不会取代程序员，但会用AI的程序员会取代不会用的。**"

这样既展示了你的行动力，又没有偏离Java工程师的定位。
